## Key Takeaways

1. **System Limit is Absolute**
   - No more than 10 calls active at any time
   - Checked BEFORE user limit
   - Applies to ALL users combined

2. **User Limit is Individual**
   - Each user respects their `concurrent_calls_limit`
   - User A at 5/5 doesn't block User C at 0/5
   - BUT: System limit still applies globally

3. **Priority System Works**
   - Direct calls (100) always processed before campaign calls (0)
   - Within same priority: FIFO (first in, first out)
   - Queue processor checks priority with: `ORDER BY priority DESC, created_at ASC`

4. **Automatic Processing**
   - Queue processor runs every 10 seconds
   - Allocates slots as they become available
   - Round-robin ensures fairness among users

5. **No Race Conditions**
   - All checks use atomic CTE queries
   - Slot reservation is transactional
   - No double-booking possible
